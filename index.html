Lógica de Programação

O que é a lógica de programação?

A lógica de programação é o uso da lógica para criação de um código, ou seja, realizar o passo a passo de uma ação que será executada pelo código. Esta ferramenta é muito importante pois é através do entendimento dela que se torna possível criação e formatação de um código funcional e responsivo. Mas como que podemos compreender a lógica por detrás dos programas, ou melhor, dos computadores?

Para isso podemos pensar que os primeiros computadores foram criados para serem utilizados como calculadoras. Sim, e essa base voltada para os cálculos está incorporada nos computadores atuais. Mas hoje em dia os computadores fazem muitas funções além de cálculos, eles conseguem realizar escolhas situacionais, cálculos extremamente rápidos e complexos, traduzir imagens e sons para códigos e vice versa. E tudo isso só é possível devido à códigos responsívos e complexos que utilizam muita lógica de programação para funcionarem em conjunto.

Existe ainda uma forma de armazenamento de dados nos computadores, que possibilita a reutilização destes dados no código de forma mais prática. Uma maneira de efetuarmos este armazenamento é através das variáveis. A versatilidade das variáveis é tamanha que podemos configurar equações complexas e até mesmo criarmos fórmulas com elas, de maneira que consigamos obter os resultados esperados e solucionar os problemas.

    Tipos de dados

    Na programação podemos utilizar diferentes tipos de dados. A variável acima que vimos utiliza dados numéricos, mas em totalidade existem outros tipos de dados. Vamos ver alguns deles:

  • Dados booleanos: os dados booleanos sempre retornarão uma afirmativa, ou seja, dirão se um determinado intervalo é verdadeiro (true) ou falso (false). A utilização dos dados booleanos nos permite executar determinada ações a depender da veracidade de um intervalo, ou seja, permite que o programa tome uma decisão dependendo das ações, ou condições, precedentes. E isto é extremamente interessante para a programação pois possibilita a criação de um código mais limpo mesmo que tenha uma complexidade grande.
  • Números inteiros: Estes são os números sem fração, como por exemplo 1, 2, 3, 4...
  • Números fracionários: São constituídos dos números acrescids de ponto, como por exemplo 1.2, 1.1, 1.3, 3.1, ...
  • Texto: E o texto que é constituído por todo e qualquer texto que você queira utilizar como dado.

Podemos ainda comparar operadores booleanos, mas para isso precisamos utilizar alguns operadores especiais que nos possibilitam estas comparações e estes operadores são:

    O operador || tem uma dominância do operador "true", desta forma todas as comparações entre true e false que contiverem um valor verdadeiro (true) irão ser dadas como verdadeiras. Veja como fica:

    false || false = false
    true || false = true
    false || true = true
    true || true = true

    Já em contrapartida do operador ||, o operador && irá retornar false sempre que um valor for falso. Ou seja, só retornará true quando TODOS os valores operados forem verdadeiros. Veja como fica:

    false && false = false
    true && false = false
    false && true = false
    true && true = true

    Vale ressaltar que a operador && tem prioridade sobre o ||, então em uma comparação onde se utiliza || e && o operador && será processado antes que os operadores ||. Mas é possível forçar a precedência do || utilizando ( ) para isso. Veja abaixo:

    Já o operador ! tem como função apenar exercer a inversão da lógica, ou seja, se o resultado seria "true" com o ! ele responde "false" e vice versa. Podemos inverter comparações numéricas também, por exemplo:

    !(10<20) = false

    Desta forma teremos o inverso, acusando que o número 10 não é menor que 20. Este detalhe pode ser muito útil ao criarmos estruturas de decisão, pois poderemos induzir o computador a ler trechos do código com inversão lógica, possibilitando a criação de maiores tomadas de decisão dentro dos nossos códigos.

    Os comparadores são a base das estruturas de decisão, isto significa que é através dos operadores de comparação lógica que é possível criar as tomadas de decisão dos programas. Estas toamdas de decisão possibilitam que o programa se torne mais autônomo e passe a impressão de inteligência quando seu código for bem elaborado.
